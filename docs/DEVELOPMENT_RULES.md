# 開発・デプロイルール (Development & Deployment Rules)

このプロジェクトでは、品質担保と意図しない変更を防ぐため、以下のプロセスルールを遵守します。

## 1. 実装ルール (Implementation Rules)
*   **ローカル優先**: 機能追加や修正は、まずローカル環境 (`localhost`) で実装・検証を行います。
*   **挙動確認**: 実装が完了したら、必ず**ユーザーにローカル環境での挙動確認を求めます**。
    *   スクリーンショットや具体的な操作手順を提示し、期待通りの動作をしているか確認してもらいます。

## 2. デプロイルール (Deployment Rules)
*   **デプロイ確認**: ユーザーによるローカル検証が完了（OKが出た）した後、「デプロイを行っても良いか」をユーザーに確認します。
*   **デプロイ実行**: ユーザーからの**明示的な承諾（OK）**を得てから、デプロイ作業（`git push` 等）を開始します。
    *   勝手なデプロイは禁止です。
