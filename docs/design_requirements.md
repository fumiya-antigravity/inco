# Backlog風タスク管理アプリ デザイン要件定義書

## 1. プロダクト概要

### コンセプト: "Friendly Efficiency"（親しみやすい効率化）
- 開発者だけでなく、非エンジニア（営業、事務など）も直感的に使える、モダンで親しみやすいUIを目指す。
- Asanaのような柔軟性と、Backlogのような分かりやすさを融合させる。

### ターゲット
- ソフトウェア開発チーム
- Web制作会社
- 一般企業の部署（総務・営業など）

## 2. デザイン原則 (Design Principles)

### 2.1. 形状 (Shape) & スタイル
- **角丸 (Rounded Corners)**:
    - **コンテナ・外枠**: `rounded-xl` (12px) 以上を使用し、柔らかく包容力のある印象を与える。
    - **ボタン・入力フォーム**: `rounded-lg` (8px) を基準とし、クリックしやすさと親しみやすさを両立する。
    - **アイコン**: 円形 (`rounded-full`) または `rounded-lg` を使用。
- **余白 (Spacing)**:
    - 情報は詰め込みすぎず、適度な余白 (`p-4`, `gap-4` 等) を持たせる。
    - 特に詳細パネルやリストの行間は広めに取り、視認性を高める。
- **インタラクション (Interaction)**:
    - **ホバーエフェクト**: 要素にマウスオーバーした際、背景色を薄く変化させたり (`hover:bg-slate-100`)、アイコンを表示させたりして、操作可能性を示唆する。
    - **トランジション**: 色や配置の変化には `transition-all duration-200` 程度のアニメーションを付け、滑らかな操作感を提供する。

### 2.2. カラーパレット (Color Palette)
Tailwind CSSのカラーパレットをベースに使用。

- **プライマリカラー**: `Emerald` (500/600)
    - 信頼、安心、完了、進行を象徴する緑色。
    - 主なアクションボタン、完了チェック、強調表示に使用。
- **アクセントカラー**: `Blue`, `Indigo`, `Orange`, `Rose`
    - プロジェクトアイコン、優先度、ステータスなどの識別に使用。

#### ベースカラー (Light Mode)
- **背景**: White (`#FFFFFF`) / Slate-50 (`#F8FAFC`)
- **テキスト**: Slate-800 (メイン), Slate-500 (サブ)
- **ボーダー**: Slate-200

#### ベースカラー (Dark Mode)
- **サイドバー**: Slate-900 (`#0F172A`) - 濃い青灰色
- **メイン背景**: Black (`#000000`) または Slate-950
- **パネル・カード背景**: Zinc-900 / Slate-800 - 階層を表現するための少し明るい黒
- **テキスト**: Slate-200 (メイン), Slate-400 (サブ)
- **ボーダー**: Zinc-800 / Slate-700

## 3. UI構造 (Layout Structure)

### 3.1. サイドバー (Sidebar)
- **配置**: 画面左側に固定。
- **機能**:
    - **上部**: ハンバーガーメニューによる開閉（幅切り替え）。
    - **中部**: 「ホーム」「マイタスク」「受信トレイ」などのグローバルナビゲーションと、プロジェクト一覧（折りたたみ可能）。
    - **下部**: アプリロゴ、組織名、ダークモード切り替えスイッチ。
- **省略表示**: 閉じた状態では幅を縮小し、アイコンのみを表示して作業領域を広げる。

### 3.2. メインエリア (Main Content)
- **ヘッダー**:
    - パンくずリストや現在のビュー名（プロジェクト名）を表示。
    - 検索バー、通知アイコン、ユーザーアイコンを配置。
- **タブナビゲーション**:
    - 「概要」「リスト」「ボード」「Wiki」などのビュー切り替えタブ。
- **ツールバー**:
    - タスク追加ボタン、フィルター、ソートなどの操作系を配置。
    - Wikiタブなど、ビューによっては非表示にする。
- **コンテンツ表示**:
    - **ボードビュー**: カンバン方式。セクション（カラム）ごとにタスクカードを表示。DnD対応。
    - **リストビュー**: テーブル方式。セクションごとのヘッダーを持ち、行形式でタスクを表示。DnD対応。
    - **Wikiビュー**: ドキュメント閲覧・編集エリア。

### 3.3. タスク詳細パネル (Task Detail Panel)
- **表示形式**: 画面右側からスライドインまたはオーバーレイ表示（フローティング）。
    - 背景のメインエリアをクリック可能にする（非モーダル）。
    - 幅は可変（レスポンシブ）、PCでは 550px 程度を基準とする。
- **ヘッダー**:
    - 完了ボタン（左上）、閉じるボタン（右上）、その他の操作メニュー。
- **コンテンツ**:
    - **タイトル**: 大きな文字で編集可能。
    - **プロパティ**: 担当者、期限日、プロジェクト、セクション、ステータス、優先度、種別などをグリッド状に配置。
        - 値がある場合は背景色を付けて強調。
        - 値がない場合は薄く表示し、ホバーで入力可能であることを示す。
    - **詳細 (Description)**: テキストエリア（Markdown対応を想定）。
    - **サブタスク**: チェックリスト形式。インライン追加が可能。
    - **アクティビティ**: 変更履歴とコメントを時系列（新しい順）で表示。コメント入力欄を持つ。

## 4. コンポーネント要件 (Component Requirements)

### 4.1. ボタン (Buttons)
- **Primary**: 背景色あり（Emerald）、白文字。主要なアクション（新規作成など）。
- **Secondary**: 背景透明または薄いグレー、枠線なし。補助的なアクション（キャンセル、編集など）。
- **Icon Button**: アイコンのみ。ホバー時に背景色変化。

### 4.2. 入力フォーム (Inputs)
- **インライン編集**: テキストをクリックすると即座に入力モードに切り替わる。
- **リアルタイム保存**: 保存ボタンを押さずとも、入力・変更が即座にデータに反映される体験（UX）を重視。
- **プレースホルダー**: 入力がない場合は薄い文字でガイドを表示。

### 4.3. セレクター (Dropdowns/Popovers)
- ステータス、優先度、種別などの選択に使用。
- クリックすると選択肢がポップオーバーで表示される。
- 現在の値に応じた色分け（例: 高優先度=赤、完了=緑）を行う。

## 5. インタラクション要件 (Interaction Requirements)

### ドラッグ＆ドロップ (DnD)
- ボードのカラム間、リストの行間でタスクを移動できること。
- 移動によりセクション（ステータスではなくグループ）が更新されること。

### 詳細パネルの開閉
- タスクをクリック -> 詳細パネルが開く。
- 別のタスクをクリック -> 詳細パネルの内容が切り替わる（パネルは開いたまま）。
- プロジェクトを切り替える -> 詳細パネルは閉じる。
- 閉じるボタンまたは Esc キー -> 詳細パネルが閉じる。
